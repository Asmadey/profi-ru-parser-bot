# Profi.ru Parser Bot

Telegram-бот, который автоматически мониторит бэкофис Profi.ru на предмет новых заказов, связанных с разработкой ИИ/ML/чат-ботов/автоматизацией, и отправляет отфильтрованные уведомления в Telegram-чат.

<!-- ![Пример работы](docs/screenshots/parser_example.jpg) -->

## Возможности

- Парсинг ленты заказов Profi.ru через Playwright + Chrome DevTools Protocol (CDP)
- Фильтрация по 35+ ключевым словам (ИИ, ML, чат-боты, парсеры, CRM, автоматизация — RU + EN)
- Отсев нерелевантных тем (таргетинг/реклама/SMM) и запрещённых платформ (Instagram, WhatsApp, Facebook, Discord)
- Проверка бюджета ≥ 10 000 ₽
- Дедупликация заказов (seen_ids)
- Отправка уведомлений в Telegram через SOCKS5-прокси
- Ручной логин на Profi.ru при первом запуске с сохранением сессии

## Архитектура

Проект работает как единый процесс `run_all.py` (точка входа), который запускает два параллельных компонента:

```
                     run_all.py  (точка входа — оркестратор)
                    ┌──────┴──────────────────────┐
                    │                             │
          supervise_parser()              telegram_notifier()
          (asyncio task)                  (asyncio task)
                    │                             │
                    ▼                             ▼
         ┌─────────────────┐          ┌──────────────────────┐
         │  main.py        │          │  run_all.py           │
         │  (subprocess)   │          │  (notifier loop)      │
         │                 │          │                      │
         │  Playwright     │          │  Читает new_orders    │
         │  ┌── CDP ──►    │          │  .jsonl через курсор  │
         │  │  Chrome:9225 │          │  (bot_cursor.json)    │
         │  │              │          │                      │
         │  │ auth.py      │          │  format_order()       │
         │  │  (логин при  │          │  → tg_formatter.py    │
         │  │   первом     │          │                      │
         │  │   запуске)   │          │  bot.send_message()   │
         │  │              │          │  через SOCKS5-прокси  │
         │  │ client.py    │          │                      │
         │  │ (ProfiClient)│          │  Flood control        │
         │  │              │          │  (TelegramRetryAfter) │
         │  │ parser.py    │          └──────────────────────┘
         │  │ (DOM parse)  │
         │  │              │
         │  │ filters.py   │
         │  │  (фильтрация │
         │  │   по ключам) │
         │  │              │
         │  │ storage.py   │
         │  │  (seen_ids)  │
         │  └─────────────│
         │                 │
         │  new_orders.jsonl ────► (общий файл между процессами)
         └─────────────────┘
```

### Поток данных

1. **`run_all.py`** запускает два параллельных asyncio-компонента.
2. **Парсер** (сабпроцесс `main.py`):
   - Подключается к внешнему Chrome через CDP (порт 9225)
   - Загружает `storage_state.json` для авторизованной сессии
   - Опрашивает `https://profi.ru/backoffice/` с человекоподобными задержками (45±25 c)
   - Извлекает карточки заказов из DOM (`parser.py`)
   - Фильтрует через `filters.py` (ключевые слова + бюджет)
   - Дедуплицирует через `seen_ids.json`
   - Дописывает подходящие заказы в `new_orders.jsonl`
3. **Нотификатор** (в `run_all.py`):
   - Каждые 3 секунды читает `new_orders.jsonl` через байтовый курсор (`bot_cursor.json`)
   - Форматирует заказ через `tg_formatter.py`
   - Отправляет сообщение в Telegram через SOCKS5-прокси
   - Соблюдает flood-контрол (`TelegramRetryAfter`)

## Требования

- **Python 3.10+**
- **Chrome / Chromium** с удалённой отладкой на порту 9225 — должен быть запущен **до** старта бота:

  ```bash
  google-chrome --remote-debugging-port=9225 --user-data-dir=/tmp/chrome-profi
  ```

  Бот подключается к уже запущенному браузеру через CDP; сам он Chrome не запускает.

## Установка

```bash
# 1. Клонировать репозиторий
git clone <repo-url> Profi.ru_Parser_Bot
cd Profi.ru_Parser_Bot

# 2. Установить зависимости
pip install -r requirements.txt

# 3. Установить браузеры Playwright (для underlying-движка)
playwright install

# 4. Создать .env из примера
cp .env.example .env
```

### Переменные окружения (`.env`)

| Переменная | Описание |
|---|---|
| `BOT_TOKEN` | Токен Telegram-бота от [@BotFather](https://t.me/BotFather) |
| `ADMIN_CHAT_ID` | ID чата, куда отправляются уведомления (ваш личный chat ID) |
| `TELEGRAM_PROXY` | SOCKS5-прокси для подключения к Telegram API (по умолчанию `socks5://127.0.0.1:10808`) |

## Запуск

```bash
python run_all.py
```

### Первый запуск

При первом запуске Chrome открывается в **non-headless** режиме — необходимо вручную войти в аккаунт Profi.ru. После авторизации сессия сохраняется в `storage_state.json`, и последующие запуски используют сохранённую сессию (headless-режим можно включить в `config.py`).

## Фильтрация заказов

Заказ проходит в уведомления, если выполняются **все** условия:

| Критерий | Логика |
|---|---|
| **Целевые ключевые слова** | 35+ паттернов (RU+EN): бот/чат-бот/bot/chatbot, CRM/ЦРМ/СРМ, парсер/parse, автоматизация/automation и др. |
| **Намерение разработки** | Слова-маркеры: разработка, создать, написать, настроить, внедрить, интеграция, требуется, нужен и т.д. |
| **Запрещённые темы** | Отсев: таргет, таргетинг, контекстная реклама, SMM, продвижение, настройка рекламы |
| **Запрещённые платформы** | Отсев: Instagram, WhatsApp, Facebook, Discord (и русские транслитерации) |
| **Бюджет** | ≥ 10 000 ₽ (если бюджет указан; если не указан — пропускается) |

## Структура проекта

| Файл | Назначение |
|---|---|
| `run_all.py` | Точка входа: оркестратор парсера + Telegram-нотификатор |
| `main.py` | Сабпроцесс парсера: цикл опроса Profi.ru |
| `client.py` | `ProfiClient` — обёртка над Playwright-страницей |
| `auth.py` | Авторизация на Profi.ru (ручной логин + сохранение состояния) |
| `parser.py` | Извлечение карточек заказов из DOM |
| `filters.py` | Фильтрация по ключевым словам, бюджету, теме и платформам |
| `storage.py` | Хранение `seen_ids.json` (дедупликация) |
| `tg_formatter.py` | Форматирование заказа в Telegram-сообщение |
| `logger_setup.py` | Настройка логирования |
| `config.py` | Конфигурация (URL, таймауты, пути файлов, селекторы) |
| `.env.example` | Шаблон переменных окружения |

## Runtime-артефакты

Эти файлы и директории создаются во время работы бота (не входят в репозиторий):

| Артефакт | Назначение |
|---|---|
| `storage_state.json` | Сохранённая сессия Profi.ru (cookies, localStorage) |
| `seen_ids.json` | Множество ID уже обработанных заказов |
| `new_orders.jsonl` | Лог прошедших фильтр заказов (по строкам JSON) |
| `bot_cursor.json` | Байтовый курсор чтения `new_orders.jsonl` нотификатором |
| `logs/` | Директория с логами парсера и отладочными дампами |
